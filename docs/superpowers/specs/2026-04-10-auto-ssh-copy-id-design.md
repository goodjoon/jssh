# Auto ssh-copy-id Suggestion Design

## 1. 개요
SSH 키 인증 방식을 사용하여 서버 접속을 시도할 때, 서버 측에 내 공개키(Public Key)가 등록되어 있지 않아 `Permission denied (publickey)` 에러가 발생하는 경우, 스크립트가 이를 감지하고 자동으로 `ssh-copy-id` 실행을 제안하는 편의 기능을 추가한다.

## 2. 동작 로직 (`libs/ssh_utils.sh`)
- 패스워드가 없는 (SSH 키 기반) 연결 시도 직전, `ssh -o BatchMode=yes ... true` 명령을 통해 인증 성공 여부를 빠르게 사전 검사(Pre-flight Check)한다.
- 검사에 성공하면, 원래 의도대로 `exec ssh` (또는 `command ssh`)를 통해 실제 터미널 세션으로 진입한다.
- 검사에 실패하고 에러 메시지에 `Permission denied (publickey`가 포함된 경우:
  - 사용자에게 `gum confirm` 창을 띄워 "서버에 내 공개키를 등록하시겠습니까?" 라고 제안한다.
  - "예"를 선택하면, `ssh-copy-id` 명령어를 구성하여 실행한다.
    - 지정된 `$ssh_key`가 있다면 `-i "${ssh_key}.pub"` 형태로 공개키 경로를 명시한다.
    - 공개키 파일이 로컬에 실제로 존재하는지 사전에 확인한다.
  - `ssh-copy-id` 과정에서 한 번의 서버 패스워드 입력이 요구되며, 성공 시 즉시 `exec ssh`로 연결을 이어간다.
  - 거부하거나 복사에 실패하면 접속을 중단하고 메뉴로 복귀한다.

## 3. UI 및 사용자 안내
- **실패 감지:** `❌ 키 인증 실패: 서버에 공개키가 등록되지 않았습니다.`
- **제안 다이얼로그:** `지금 서버에 내 공개키를 등록(ssh-copy-id) 하시겠습니까?`
- **승인 후:** `ℹ️ 원격 서버의 패스워드를 한 번 입력해 주셔야 합니다.`
- **성공 후:** `✅ 공개키 등록 성공! 접속을 시도합니다...`

## 4. 예외 및 환경 고려사항
- `ssh_key` 옵션이 지정되었으나 해당하는 `.pub` 공개키 파일이 없는 경우, 안내 메시지를 띄우고 `ssh-copy-id`를 생략한다.
- `gum`이 시스템에 설치되지 않은 경우 호환성을 위해 에러 메시지만 출력하고 스크립트를 종료한다.
