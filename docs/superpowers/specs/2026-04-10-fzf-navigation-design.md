# FZF Navigation Enhancement Design

## 1. 개요
`jssh-gum`의 메인 서버 목록 화면(`fzf` 기반)에서 서버 개수가 많아질 때 방향키 조작만으로 위아래 이동이 오래 걸리는 불편함을 해소하기 위해 키보드 네비게이션 기능을 개선합니다.

## 2. 주요 개선 사항
1. **무한 스크롤 (Wrap-around):**
   - 첫 번째 항목에서 `Up` 방향키를 누르면 목록의 가장 마지막 항목으로 이동합니다.
   - 마지막 항목에서 `Down` 방향키를 누르면 첫 번째 항목으로 이동합니다.
2. **페이지 이동 (Page Up / Page Down):**
   - `Left` 방향키를 누르면 이전 페이지(Page Up)로 이동합니다.
   - `Right` 방향키를 누르면 다음 페이지(Page Down)로 이동합니다.

## 3. 구현 방식
- `jssh-gum` 내의 `list_servers()` 함수에서 `fzf`를 호출하는 부분을 수정합니다.
- `fzf` 명령어에 다음과 같은 옵션을 추가합니다:
  - `--cycle`: 위아래 무한 스크롤(Wrap-around) 기능을 활성화합니다.
  - `--bind 'left:page-up,right:page-down'`: 좌/우 방향키를 페이지 업/다운 기능으로 매핑합니다.

## 4. 사이드 이펙트 검토
- `fzf`의 기본 동작에서 좌/우 방향키는 검색어(프롬프트) 입력 창에서의 커서 이동 역할을 할 수 있습니다. 하지만 `fzf` 특성상 입력창의 텍스트가 짧을 때는 페이지 이동으로 사용해도 무방하며, 대부분의 사용자는 서버 탐색용으로 방향키를 많이 사용하므로 편의성이 훨씬 증가합니다.
- (참고) 검색 프롬프트에서 글자를 수정할 때는 Backspace 나 타이핑을 주로 사용하게 됩니다.

## 5. 테스트 계획
- `jssh-gum` 실행 후 서버 목록에서 첫 항목에서 위 화살표 입력 시 마지막 항목으로 이동하는지 확인.
- 목록이 화면을 넘어갈 정도로 충분히 있을 때 좌/우 화살표 입력 시 한 화면(페이지) 단위로 스크롤 되는지 확인.
