# SSH Key 신규생성 (Ctrl-G) 기능 설계

## 1. 개요
사용자가 서버 정보 입력(추가/편집) 폼에서 SSH 키를 지정할 때, 기존 키를 입력하는 대신 터미널 내에서 곧바로 새로운 SSH 키를 생성하여 적용할 수 있도록 한다. 단축키 `Ctrl-G`를 사용하여 생성 마법사를 호출한다.

## 2. `libs/form.py` 변경 (UI 및 이벤트 처리)
- **UI 표시**: SSH 키 필드 우측 또는 폼 하단 힌트 영역에 `[Ctrl-G: 신규생성]` 안내 문구를 추가한다.
- **이벤트 처리**: 
  - `get_wch()` 루프 내에서 `Ctrl-G` (키 코드 7) 입력을 감지한다.
  - `Ctrl-G`가 눌리면 현재 폼에 입력 중이던 값들을 유지하기 위해 표준 출력(stdout)으로 현재 값들을 출력하고, 구분을 위한 특별한 **종료 코드(exit code 7)**로 스크립트를 종료한다.

## 3. `jssh-gum` 메인 스크립트 변경 (폼 루프 및 마법사 연동)
- **폼 호출 루프화**:
  - `add_server` 및 `edit_server` 내의 `form.py` 호출을 `while true` 루프로 감싼다.
  - `form.py`의 종료 코드가 `0`(저장)이면 루프를 탈출하고 파일에 저장한다.
  - `form.py`의 종료 코드가 `1`(취소)이면 즉시 함수를 빠져나간다.
  - `form.py`의 종료 코드가 `7`(신규생성)이면 현재까지 입력된 폼 데이터 변수들을 업데이트한 뒤 **SSH 키 생성 마법사**를 실행한다.
- **SSH 키 생성 마법사 (`generate_ssh_key_wizard` 함수)**:
  1. `gum input`을 사용하여 저장 경로를 입력받는다. (기본값: `~/.ssh`)
  2. `gum input`을 사용하여 파일 이름을 입력받는다.
  3. `gum input`을 사용하여 SSH 접속 ID(코멘트용)를 입력받는다.
  4. 입력된 저장 경로 디렉토리가 존재하는지 확인하고, 없다면 `mkdir -p`로 생성한다.
  5. `ssh-keygen -t ed25519 -C "접속 ID" -f "경로/파일이름" -N ""` 명령으로 키를 생성한다.
  6. 생성된 전체 경로를 `new_ssh_key` 변수에 덮어씌운다.
- **폼 재호출**: 
  - 마법사가 끝나면 루프가 반복되며, 갱신된(생성된 키 경로가 포함된) 변수값들로 `form.py`가 다시 열려 사용자에게 보여진다.

## 4. 기대 효과
터미널 환경의 제약을 벗어나 폼 입력 도중 상태를 잃지 않고 다른 상호작용(키 생성)을 다녀올 수 있으며, 생성된 결과를 즉각 폼에 반영하는 유연한 UX를 제공한다.
