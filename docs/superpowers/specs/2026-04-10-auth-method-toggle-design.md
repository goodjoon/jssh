# Auth Method Toggle Design for jssh-gum

## 1. 개요
jssh-gum의 접속 정보 편집 화면(`libs/form.py`)에서 서버 접속 시 사용할 인증 방식을 명확하게 선택할 수 있도록 '인증 방식' 토글(Toggle) 필드를 추가합니다. 사용자는 '패스워드' 방식과 'SSH 키' 방식을 명시적으로 선택할 수 있으며, 선택된 방식에 따라 저장되는 정보가 결정됩니다.

## 2. UI 설계 (`libs/form.py`)
- 기존 텍스트 입력 필드 외에 "선택형(Toggle)" 필드를 지원하도록 `form.py`를 확장합니다.
- 폼 인자 규격 확장: `label value type` 형식에서 `type` 파라미터에 기존 `0`(평문), `1`(패스워드) 외에 `choice:옵션1,옵션2` 형태를 추가로 지원합니다.
  - 예시: `"인증방식" "패스워드" "choice:패스워드,SSH 키"`
- 선택형 필드 동작:
  - 해당 필드에 커서가 위치했을 때 일반 문자열 타이핑이나 삭제(Backspace/Delete) 조작은 무시됩니다.
  - 좌/우 방향키 또는 스페이스바를 누르면 제공된 옵션 목록 내에서 값이 순환(Toggle) 변경됩니다.
- 렌더링: 선택형 필드 값의 양옆에 화살표(예: `◀ 패스워드 ▶`)를 표시하여 좌우 조작이 가능함을 시각적으로 나타냅니다.

## 3. 데이터 흐름 및 저장 로직 (`jssh-gum`)
- **서버 추가/편집 폼 구성:**
  - 폼 하단에 `사용자`, `인증방식`, `패스워드`, `SSH키` 순으로 필드를 배치합니다.
  - 편집 모드 진입 시 `인증방식` 필드의 초기값은 기존 데이터 상태에 따라 결정됩니다.
    - `ssh_key` 데이터가 존재하면 "SSH 키"를 초기값으로 설정합니다.
    - 그렇지 않으면 "패스워드"를 초기값으로 설정합니다.
- **저장(Sanitize) 로직:**
  - 폼에서 반환된 `인증방식` 값을 확인하여 유효하지 않은 데이터를 초기화(Sanitize)한 후 저장합니다.
  - **"패스워드" 선택 시:** 반환된 `SSH키` 값을 강제로 비우고 패스워드만 `servers.list`에 유지/저장합니다.
  - **"SSH 키" 선택 시:** 반환된 `패스워드` 값을 강제로 비우고 SSH 키 경로만 `servers.list`에 유지/저장합니다.
  - `servers.list` 파일 포맷(8개 필드: `별칭|구분|호스트|IP|포트|사용자|패스워드|SSH키`)은 그대로 유지됩니다.

## 4. 하위 호환성
- `servers.list` 데이터 구조는 변경되지 않으므로 기존 데이터 목록과 완벽히 호환됩니다.
- `form.py`의 인자 규격 확장도 하위 호환성을 유지합니다. (기존 스크립트에서 `0`이나 `1`로 호출하는 경우 영향 없음)

## 5. 예외 처리 및 고려사항
- `form.py`에서 `choice` 타입의 초기값이 옵션 목록에 없는 경우, 목록의 첫 번째 옵션을 기본값으로 강제 할당합니다.
- 폼 화면 내에 "방향키/스페이스바로 선택 가능"하다는 힌트 문구를 하단 도움말에 추가 반영합니다.

## 6. 테스트 계획
- `form.py` 단독 실행 시 `choice` 필드가 정상적으로 렌더링되고 조작되는지 테스트.
- 서버 추가 시 '인증방식'을 변경하며 저장했을 때, 선택되지 않은 방식의 데이터가 비워져서 `servers.list`에 저장되는지 검증.
- 서버 편집 진입 시 기존 설정(패스워드 사용 or SSH 키 사용)에 맞게 '인증방식' 토글이 올바른 상태로 초기화되는지 확인.
