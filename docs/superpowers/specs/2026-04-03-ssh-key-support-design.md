# jssh-gum SSH Key Support Design

## 1. 개요
현재 패스워드 기반 및 설정파일 의존적인 jssh-gum의 SSH 접속 기능에, 사용자 지정 SSH Private Key(`-i` 옵션)를 명시적으로 설정하여 접속할 수 있는 기능을 추가한다. 

## 2. 데이터 구조 변경 (`servers.list`)
- 기존 7개 필드 구조에 8번째 필드로 `ssh_key`를 추가한다.
- 포맷: `별칭|구분|hostname|ip|port|user|password|ssh_key`
- 하위 호환성: 기존 7개 필드만 있는 데이터도 정상 동작하도록 파싱 로직을 개선한다.

## 3. 화면 및 UI 변경 (`jssh-gum`)
- **서버 목록 화면 (fzf)**
  - 마지막 컬럼에 `SSH 키` 컬럼을 추가한다.
  - 경로가 길 경우 UI가 깨지지 않도록 Python 스크립트에서 너비를 제한(예: 15~20자)하고 말줄임 처리한다.
- **서버 추가/편집 폼 (`libs/form.py` 연동)**
  - 폼의 8번째 필드로 `SSH 키 경로` 입력란을 추가한다.
  - 힌트 메시지에 `예: ~/.ssh/id_rsa` 와 같은 안내를 추가한다.
- **연결 확인 화면 (`connect_to_server`)**
  - 접속 전 출력되는 정보에 "SSH 키: <입력된경로>" 를 표시한다.
  - 접속 실행 명령어 미리보기에도 `-i <경로>`가 포함되어 보이도록 한다.

## 4. 접속 로직 변경 (`libs/ssh_utils.sh`)
- `connect_ssh` 및 `connect_ssh_exec` 함수의 파라미터 시그니처를 수정하여 `ssh_key`를 전달받도록 한다.
- `ssh_key` 파라미터 값이 존재할 경우, `ssh_args` 배열에 `-i "$ssh_key"` 옵션을 추가하여 `ssh` 명령이 해당 키를 사용하도록 구현한다.

## 5. 테스트 계획
- 기존(7개 필드) 데이터가 오류 없이 로드되는지 확인
- 서버 추가 폼에서 SSH 키를 입력하고 저장했을 때 `servers.list`에 8번째 필드로 저장되는지 확인
- 서버 편집 폼에서 기존 SSH 키 값이 정상적으로 로드되고 수정되는지 확인
- fzf 목록에서 SSH 키 컬럼이 올바른 위치에, 적절히 잘려서(말줄임) 표시되는지 확인
- 실제 연결 시 `-i` 옵션이 정상적으로 넘어가서 접속이 이루어지는지 확인
